# 标准交付清单

本文档用于把项目从“能演示”推进到“能交付、能报价、能验收”。

## 1. 推荐交付包结构

建议对外交付时至少拆成 5 类资产：

### 1.1 产品运行包

- 前端构建产物
- 后端运行代码
- 配置文件模板
- 数据库初始化脚本
- 启动脚本 / 服务脚本

### 1.2 文档包

- `README.md`
- `docs/deployment.md`
- `docs/sales-demo-playbook.md`
- `docs/commercial-boundary.md`
- 本文档

### 1.3 演示与销售包

- 销售首页演示口径
- 老板驾驶舱演示顺序
- 套餐报价承接页
- 样板场景与默认数据说明

### 1.4 配置与规则包

- `config/products.json`
- `config/sites.json`
- `config/runtime.json`
- `config/alerts.json`

### 1.5 验收与支持包

- 验收标准
- 问题反馈入口
- 售后支持窗口
- 升级与迭代边界说明

## 2. 推荐标准交付内容

### A. 必交

- 可运行前后端项目
- 部署说明
- 环境变量示例
- 数据库初始化方式
- 演示说明
- 套餐与报价说明
- 商用边界说明

### B. 建议交

- 默认演示数据或样板场景
- 常见问题 FAQ
- 培训讲义
- 验收用测试路径

### C. 高价项目建议额外交

- 私有化部署手册
- 品牌化改造说明
- 自定义规则配置说明
- 运维巡检清单
- 版本升级说明

## 3. 实施前检查

交付前建议确认：

1. 目标环境是 Windows、Linux 还是容器。
2. 数据库使用 SQLite 还是 MySQL。
3. 是否允许联网抓取。
4. 是否需要外部 AI 能力。
5. 是否需要离线演示模式。
6. 是否需要品牌化名称、Logo、主题替换。

## 4. 推荐验收路径

建议把验收路径写成客户能看懂的业务语言，而不是开发语言。

### 路径一：销售演示开场

- 打开系统默认进入销售首页
- 点击按钮进入老板驾驶舱
- 页面可看到今日重点、机会和风险摘要

### 路径二：经营信号说明

- 查看重点商品信号
- 能看到推荐动作、原因、置信度
- 能说明为什么建议立即采购 / 分批采购 / 延后采购

### 路径三：菜单采购增强

- 输入菜单或导入菜单
- 生成采购建议
- 能看到市场建议、风险说明、推荐动作

### 路径四：报价承接

- 查看套餐信息
- 能区分演示版、经营决策版、定制版
- 能自然进入报价讨论

## 5. 交付责任建议拆分

### 产品侧

- 保证系统可运行
- 保证核心演示路径可用
- 保证新增接口与基础测试通过

### 客户侧

- 提供部署环境
- 提供可用的数据来源与合规授权
- 配合确认品牌、组织和使用范围

### 双方共担

- 规则调优
- 演示话术适配
- 交付后运营方式

## 6. 当前版本可直接引用的验收事实

当前仓库可直接用于交付说明的事实包括：

- 后端经营信号、采购增强建议、销售演示与套餐接口已接入
- 前端已支持销售首页、老板驾驶舱、经营信号页和套餐承接
- 前端生产构建已通过
- 后端接口测试已通过
- Playwright 移动端回归已通过

## 7. 建议交付状态分级

### 可演示

- 能跑起来
- 能走完整销售路径
- 有默认演示内容

### 可交付

- 有部署文档
- 有配置说明
- 有验收路径
- 有商用边界说明

### 可规模化出售

- 有明确授权模式
- 有标准报价梯度
- 有交付清单
- 有售后与升级边界
- 有第三方依赖和合规说明
