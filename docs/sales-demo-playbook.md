# 销售演示与报价承接说明

## 定位

当前版本不再只面向“查看价格”的操作员，而是面向三类更容易形成付费决策的角色：

- 老板 / 经营负责人：先看今天的风险、机会和一句话建议
- 采购负责人：再看单品时机、菜单采购增强建议和备选市场
- 销售 / 顾问：最后切到套餐、部署方式和交付边界

核心卖点不是“图表更多”，而是：

- 用真实多市场报价做底层事实
- 用规则优先的信号引擎给出建议
- 用销售化页面把建议转成可报价的话术和路径

## 推荐演示路径

### 1. 销售首页

目的：
- 在 30 秒内让客户知道这不是普通看表工具
- 先讲“今天该关注什么”，不是先讲字段和表格

建议话术：
- 先展示本周机会窗口、老板摘要和销售承接动作
- 不先进入复杂趋势图和全量表格

### 2. 老板驾驶舱

目的：
- 让决策者先看到风险、机会和今天建议

建议展示：
- 信号总数 / 高优先级 / 平均置信度 / 预估影响值
- 今天先看的 2 到 3 个重点事项
- 一句话摘要和成交承接建议

### 3. 经营信号与单品时机

目的：
- 证明系统不是“精装修 Demo”，而是有底层规则依据

建议展示：
- `timing_score`
- `risk_score`
- `confidence`
- `recommended_action`
- `reason_summary`

### 4. 菜单采购增强建议

目的：
- 把菜单采购从“计算价格”升级为“给出采购动作”

建议展示：
- 当前采购时机
- 主要市场与备选市场
- 风险解释
- 推荐动作

### 5. 套餐报价承接

目的：
- 在演示结束时自然进入报价，而不是停在“以后再聊”

建议展示：
- 演示成交版：快速试点、低阻力进入
- 经营决策版：主推版本，适合正式成交
- 定制交付版：用于抬高客单价和项目价值

## 新增接口

### `GET /api/signals/overview`

用途：
- 销售首页、老板驾驶舱、经营信号页共用的高层信号概览

### `GET /api/signals/{identity_key}`

用途：
- 单品时机建议、重点商品解释和趋势型说服素材

### `POST /api/procurement/recommend`

用途：
- 将菜单采购结果增强为“时机 + 风险 + 推荐动作”

### `GET /api/sales/demo-content`

用途：
- 销售首页、演示文案、证明点和讲解路径的稳定兜底内容

### `GET /api/pricing/packages`

用途：
- 报价承接、套餐对比和成交页
