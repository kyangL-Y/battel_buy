# 前端优化迭代报告 2026-06-04

## 本轮结论

本轮继续围绕首屏加载、懒加载、移动端交互、普通用户可读性和代码可维护性推进。核心结果是：登录页和移动首页不再提前加载工作台页面与业务接口，工作台数据按当前页面需要加载，前端 API 与类型模块进一步拆分，构建和回归测试已通过。

## 已完成优化

1. 首屏加载收敛
   - 移动首页保持纯入口展示，不提前请求 `/api/`。
   - 桌面首页新增回归断言，确认不提前加载工作台 chunk，也不提前请求业务接口。
   - 桌面首页即使已有登录态，也不提前拉取行情数据。

2. 数据懒加载
   - 汇总行情首批只加载当前页需要数据，后续商品通过“加载更多”触发。
   - 工作台 section 数据按当前 section 加载，减少无关接口并发。
   - 地区变化时不再无条件刷新汇总行情，只在当前视图确实需要汇总行情时请求。

3. 前端模块拆分
   - `web/src/api.ts` 保留兼容入口，HTTP 客户端、请求状态、静态快照兜底、供应商接口已拆到独立模块。
   - `web/src/types.ts` 保留兼容导出，供应商类型已拆到 `supplierTypes.ts`。
   - `xlsx` 改为按需加载，供应商导入功能不会压到首屏。

4. UI 与文案方向
   - 登录页和移动端文案已改为普通用户表达，减少专业术语和解释性废话。
   - 登录失败提示、忘记密码、显示密码按钮、输入框尺寸与移动端布局已完成调整。
   - 移动端首页、菜价、提醒、采购主流程已按 PC 端风格统一。

5. 发布产物
   - 已执行 production build。
   - `web/dist` 作为本次构建输出目录。
   - `release/dist` 已改为本地发布产物目录，不再作为 Git 跟踪资产；如需交付静态资源，应由发布流程从 `web/dist` 同步。
   - 旧 hash 资源可保留在部署环境中用于兼容线上旧页面引用，但不应继续提交进仓库。

## 本轮新增回归测试

- 桌面首页不提前加载工作台页面。
- 桌面首页有登录态时不提前加载行情数据。
- 移动首页不提前加载工作区页面。
- 移动菜价页不提前加载明细和采购页面。
- 移动工作区行情、提醒、采购三主流程可切换。
